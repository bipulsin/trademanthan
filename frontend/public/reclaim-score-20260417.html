Reclaim Probability Score

Session: 2026-04-17 (IST) · Four bought Smart Futures positions
Rule of thumb. On every scan for an open LONG/SHORT position, if price is on the wrong side of the session 15-minute VWAP (LONG below, SHORT above) we compute a 0–100 reclaim probability score. If the score is below 45 (“panic”), the Sell button on the Smart Futures Open Positions section starts blinking and the exit reason is surfaced as a hint — otherwise (“calm”) we intentionally stay patient and do not press the panic button even though price is below VWAP.

How the score is constructed

The score is computed only when scan_price is adverse relative to the live 15m session VWAP, so for a LONG position this means LTP < m15_vwap. It is a weighted blend of four components that, together, try to say “does this dip look reclaimable in the next few bars?”

score = distance (0–40) + structure (0–25) + momentum (0–15) + sector (0–20) distance = 40 × (1 − min(gap_in_ATR14 / 2.5, 1)) # shallower dip ⇒ larger structure = reward last 5m bars closing in favorable half of range + recent direction momentum = +15 if last 3 closes are monotonically favorable, else +8 if last 2 are, else 0 sector = 10 + 10 × sector_score, clamped to [0, 20]
  • Distance (0–40). Normalizes how far LTP is below VWAP in units of the 14-period ATR on the 5-minute session series. Small dips < 0.5 ATR get close to 40 points; dips > 2.5 ATR get 0.
  • Structure (0–25). Rewards the last two 5m candles closing in the favorable half of the bar range and a favorable 1-bar direction — i.e. signs of buyers absorbing the dip.
  • Momentum (0–15). Rewards monotone favorable closes in the last 3 5-minute bars.
  • Sector (0–20). Uses the existing sector_score on the row as a slow bias; symbols in weak sectors lose points.

Panic vs calm

ConditionAction
Price is above VWAP (not adverse) score N/A — no change to existing exit hints
Adverse and score ≥ 45 (calm) Do NOT panic — we actively suppress the soft 15m primary VWAP exits (15-min Close Below VWAP+Supertrend+EMA, 15-min Momentum Weak + Trend Breakdown).
Adverse and score < 45 (panic) Force exit — set exit_suggested = true with reason Below 15m VWAP — low reclaim probability (panic), which makes the Open Positions Sell button blink. Emergency / hard-stop / trailing-stop exits are always respected.

How this shows up on the live Smart Futures page

The backend now computes the reclaim score on every GET /smart-futures/daily for every bought position. When the score fires panic, two things happen on /smartfuture.html:

  • The Sell button in the Open Positions row starts blinking (class sf-btn-sell--blink). This is the existing UI for exit_suggested; the reclaim gate simply sets that flag for adverse-VWAP + low-score scans.
  • Hovering the Sell button shows the panic reason plus “Reclaim probability score: NN.N/100 (higher ≈ VWAP reclaim more likely).”

The gate is conservative by design: it never overrides Emergency, Stop-Loss or Trailing-Stop exits. It only (a) suppresses the soft 15m primary VWAP exit reasons when the score is high, or (b) forces an exit hint when the score is low and price is adverse.
