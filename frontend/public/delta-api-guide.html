Delta API User Guide

Simple guide to using the Delta API module for trading automation

Settings Delta API Guide

What is the Delta API Module?

The Delta API module is a Python tool that lets you connect to Delta Exchange and perform trading operations automatically. Think of it as a bridge between your Python scripts and the Delta Exchange trading platform.

The Delta API module provides a simple way to interact with Delta Exchange. When you create an API client, you only need to provide three basic pieces of information: your API key, API secret, and the API URL. All other details like which cryptocurrency to trade, how much to trade, and when to trade are specified when you call specific methods.

How to Use the Delta API in Your Python Scripts

Learn the basic steps to integrate the Delta API into your trading applications

Step 1: Import the Module

At the top of your Python script, you need to tell Python where to find the Delta API module. You'll add the backend folder to your Python path and then import the DeltaAPI class and TradingParams class.

Step 2: Set Up Your API Credentials

You only need to provide three basic pieces of information: your API key, API secret, and the API URL. These are the only parameters required when creating the API client.

Step 3: Create the API Client

Create the API client with just your credentials. This client will handle all communication with Delta Exchange. All other trading details like symbol, quantity, and price are specified when you call specific methods.

Step 4: Call Methods with Specific Parameters

Now you can use the client to perform specific operations. Each method requires its own parameters. For example, to get a price you need to specify which trading pair, to place an order you need to specify quantity and price.

What You Can Do with the Delta API

Here are all the different operations you can perform using the Delta API module

get_latest_price(symbol_id)

What it does: Gets the current price of any cryptocurrency pair like Bitcoin/USD or Ethereum/USD.

Parameters: symbol_id (integer) - The ID of the trading pair you want to check.

When to use: Before placing trades, to monitor price movements, or to check if prices have reached your target levels.

Response: Returns the current price as a number (like 50000.0 for Bitcoin).

get_candles(symbol_id, interval, limit)

What it does: Retrieves historical price data in the form of candlestick charts.

Parameters: symbol_id (integer), interval (string like '5m', '1h', '1d'), limit (integer for number of candles).

When to use: To analyze price trends, calculate technical indicators, or backtest your trading strategies.

Response: A list of price data including open, high, low, close prices and trading volume for each time period.

get_account_state()

What it does: Shows your account information including balances and available funds.

Parameters: None required.

When to use: Before placing trades to make sure you have enough funds, or to monitor your account status.

Response: Your account details including available balance, equity, and margin information.

place_order(symbol_id, side, order_type, qty, price, stop_loss, take_profit)

What it does: Places a new trading order (buy or sell) with optional stop-loss and take-profit levels.

Parameters: symbol_id (integer), side ('buy' or 'sell'), order_type ('market_order' or 'limit_order'), qty (integer), price (float, optional), stop_loss (float, optional), take_profit (float, optional).

When to use: When you want to buy or sell cryptocurrency automatically.

Response: Order confirmation including order ID, status, and details.

cancel_order(order_id)

What it does: Cancels a specific order that you've placed but hasn't been filled yet.

Parameters: order_id (string) - The unique identifier of the order to cancel.

When to use: When you change your mind about a trade or when market conditions change.

Response: Confirmation that the order was cancelled successfully.

get_positions()

What it does: Shows all your current open positions - trades that are still active.

Parameters: None required.

When to use: To monitor your current trades, check your profit/loss, or decide which positions to close.

Response: A list of your open positions with details like size, entry price, current profit/loss, and position ID.

close_position(position_id, qty)

What it does: Closes a specific open position by selling what you bought or buying back what you sold.

Parameters: position_id (string), qty (integer) - The amount to close.

When to use: When you want to take profits, cut losses, or close specific trades.

Response: Confirmation that the position was closed, including the final profit/loss.

edit_bracket_order(order_id, stop_loss, take_profit)

What it does: Changes the stop-loss or take-profit levels of an existing order.

Parameters: order_id (string), stop_loss (float, optional), take_profit (float, optional).

When to use: When you want to adjust your risk management levels without cancelling and replacing the order.

Response: Confirmation that the order was modified successfully.

get_orders(symbol_id, limit)

What it does: Gets a list of your open orders for a specific trading pair.

Parameters: symbol_id (integer), limit (integer, optional) - Maximum number of orders to return.

When to use: To see all your pending orders and their current status.

Response: A list of your open orders with details like order type, quantity, price, and status.

get_order_history(symbol_id, limit)

What it does: Gets a history of your completed orders for a specific trading pair.

Parameters: symbol_id (integer), limit (integer, optional) - Maximum number of orders to return.

When to use: To review your trading history, analyze performance, or for accounting purposes.

Response: A list of your completed orders with details like execution price, quantity, and timestamp.

Real-World Usage Examples

Here are some practical examples of how you might use the Delta API in different trading scenarios

Price Monitoring Bot

You could create a simple bot that checks the Bitcoin price every minute and sends you an alert when the price goes above or below certain levels. This is useful for staying informed about market movements without constantly watching the charts.

Automatic Trading Strategy

You could build a bot that automatically buys Bitcoin when the price drops to a certain level and sells it when the price rises to a target level. This way, you're buying low and selling high automatically, even when you're not at your computer.

Portfolio Rebalancing

You could create a system that automatically adjusts your cryptocurrency portfolio to maintain certain percentages. For example, if Bitcoin becomes more than 60% of your portfolio, it automatically sells some Bitcoin and buys other cryptocurrencies to bring it back to 60%.

Risk Management System

You could build a system that automatically sets stop-loss orders for all your positions. This means if any trade goes against you by more than a certain percentage, it automatically closes to limit your losses.

Market Analysis Tool

You could create a tool that analyzes price patterns and gives you trading signals. For example, it could tell you when the market is trending up or down, or when it's a good time to buy or sell based on technical indicators.

Arbitrage Bot

You could build a bot that looks for price differences between different exchanges and automatically buys on the cheaper exchange and sells on the more expensive one to make a profit from the price difference.

Tips and Best Practices

Important advice to help you use the Delta API safely and effectively

Keep Your API Keys Safe

Never share your API keys with anyone. Store them in environment variables or secure configuration files. If someone gets your API keys, they could trade with your money.

Test with Small Amounts First

Always start with small trading amounts when testing your bot. This way, if something goes wrong, you won't lose a lot of money. Once you're confident everything works, you can increase the amounts.

Always Use Stop-Loss Orders

Never place a trade without setting a stop-loss order. This automatically closes your position if the price moves against you by a certain amount, limiting your potential losses.

Don't Make Too Many API Calls

Delta Exchange has limits on how many API calls you can make per minute. If you make too many calls too quickly, your account might get temporarily blocked. Add small delays between your API calls.

Monitor Your Bot Regularly

Even though your bot runs automatically, you should check on it regularly to make sure it's working correctly. Look for any error messages or unexpected behavior.

Keep Logs of All Trades

Make sure your bot records all the trades it makes, including the reasons for each trade. This helps you understand how your strategy is performing and identify areas for improvement.

Handle Network Errors Gracefully

Internet connections can sometimes fail. Make sure your bot can handle these situations by retrying failed operations and not crashing when there are temporary network issues.

Calculate Position Sizes Carefully

Don't risk more than 1-2% of your total capital on any single trade. This way, even if a trade goes completely wrong, you won't lose a significant portion of your money.
